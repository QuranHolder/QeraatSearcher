## توثيق الجدول: `book_quran`

يمثل جدول **`book_quran`** جدول النص القرآني الأساسي المستخدم في النظام. يحتوي على نص الآيات بصيغ متعددة، ويُستخدم كمصدر مرجعي لعرض النص، والبحث، وربطه ببيانات القراءات والجداول التحليلية الأخرى.

هذا الجدول يركز على **تخزين النص الكامل للآيات** وليس اختلافات القراءات.

---

## تعريف الجدول

```sql
CREATE TABLE "book_quran" (
    "aya_index" INTEGER,
    "text" TEXT,
    "text_full" TEXT,
    "sora" INTEGER,
    "aya" INTEGER,
    "textU" TEXT
);
```

---

## الهدف من الجدول

يوفر هذا الجدول:

* النص الأساسي للآيات
* صيغ متعددة للنص لخدمة البحث والعرض
* ربط مباشر بين رقم السورة والآية والنص
* مرجعًا نصيًا لجداول القراءات مثل `quran_data`

ويُستخدم في:

* عرض المصحف داخل التطبيق
* البحث النصي
* المطابقة مع بيانات القراءات
* التحقق من سلامة البيانات

---

## أعمدة الجدول

### 1. aya_index (INTEGER)

معرّف عالمي فريد للآية.

* يُستخدم كمفتاح ربط مع الجداول الأخرى
* يسهل الوصول المباشر للآية دون الحاجة إلى (سورة + آية)

**تحسين مقترح:**

```sql
PRIMARY KEY (aya_index)
```

---

### 2. text (TEXT)

نص الآية بصيغة العرض الأساسية.

عادة يكون:

* مضبوطًا بالتشكيل
* مناسبًا للعرض داخل التطبيق

---

### 3. text_full (TEXT)

النص الكامل الموسع للآية.

قد يشمل:

* علامات الوقف
* رموز إضافية
* تنسيق طباعي خاص

يُستخدم في العرض الطباعي أو الدراسات المتقدمة.

---

### 4. sora (INTEGER)

رقم السورة التي تنتمي إليها الآية.

يُستخدم كمرجع للربط مع جدول `quran_sora`.

---

### 5. aya (INTEGER)

رقم الآية داخل السورة.

مع الحقل `sora` يشكلان تعريفًا منطقيًا للآية.

**تحسين مقترح:**

```sql
UNIQUE (sora, aya)
```

لمنع التكرار.

---

### 6. textU (TEXT)

نسخة مبسطة أو موحدة من النص.

تُستخدم عادة في:

* البحث غير الحساس للتشكيل
* المطابقة النصية
* المعالجة البرمجية

قد تكون:

* بدون تشكيل
* موحدة الحروف
* خالية من الرموز الخاصة

---

## العلاقات المنطقية

يرتبط هذا الجدول مع:

* `quran_sora` عبر الحقل `sora`
* `quran_data` عبر `aya_index` أو (sora + aya)

ويمثل المصدر النصي الأساسي للنظام.

---

## تحسينات مقترحة (اختيارية)

### 1. إضافة فهارس لتحسين البحث

```sql
CREATE INDEX idx_quran_sora_aya
ON book_quran (sora, aya);
```

```sql
CREATE INDEX idx_quran_textU
ON book_quran (textU);
```

---

### 2. فرض قيود تكامل مرجعي (إن لزم)

ربط `sora` بجدول `quran_sora` باستخدام foreign key.
