## توثيق الجدول: `shawahid_entries`

يمثل جدول **`shawahid_entries`** جدولًا مخصصًا لتخزين **مدخلات الشواهد** المرتبطة بالقراءات. يحتوي على نصوص الشواهد كما وردت في مصادرها، مع ربطها بمواضع الآيات ذات الصلة. يُستخدم هذا الجدول كمرجع علمي داعم لجدول القراءات.

يمكن اعتباره جدول **مراجع علمية نصية** مرتبطًا بنظام القراءات.

---

## تعريف الجدول

```sql
CREATE TABLE "shawahid_entries" (
    "id" INTEGER,
    "page_number" INTEGER,
    "entry_number" INTEGER,
    "source" TEXT,
    "text" TEXT,
    "subjects" TEXT,
    "aya_indices" TEXT
);
```

---

## الهدف من الجدول

يوفر هذا الجدول:

* تخزين نصوص الشواهد كما في مصادرها
* تنظيم الشواهد بحسب الصفحة والترقيم
* تحديد مصدر الشاهد
* ربط الشاهد بآية أو أكثر

ويُستخدم في:

* عرض الشواهد داخل التطبيق
* الربط العلمي مع اختلافات القراءات
* البحث في نصوص الشواهد
* الدراسات المقارنة

---

## أعمدة الجدول

### 1. id (INTEGER)

معرّف فريد لكل مدخل شاهد.

**تحسين مقترح:**

```sql
PRIMARY KEY (id)
```

---

### 2. page_number (INTEGER)

رقم الصفحة في مصدر الشواهد.

يُستخدم لتنظيم الشواهد حسب ترتيب المصدر الأصلي.

---

### 3. entry_number (INTEGER)

رقم تسلسلي للشاهد داخل الصفحة.

يسمح بوجود عدة شواهد في الصفحة نفسها.

---

### 4. source (TEXT)

رمز مختصر لمصدر الشاهد.

أمثلة من العينة:

* `ش`
* `د`

يمكن لاحقًا ربط هذه الرموز بجدول مرجعي لمصادر الشواهد.

---

### 5. text (TEXT)

النص الكامل للشاهد.

* محفوظ كما في المصدر
* قد يحتوي على علامات ضبط أو اختصارات علمية

---

### 6. subjects (TEXT)

موضوعات أو كلمات مفتاحية مرتبطة بالشاهد.

يُستخدم للتصنيف والبحث الموضوعي.

(في العينة الحالية يبدو فارغًا — يمكن استخدامه مستقبلًا للفهرسة.)

---

### 7. aya_indices (TEXT)

قائمة بمعرّفات الآيات المرتبطة بالشاهد.

تُخزن بصيغة نصية مفصولة بفواصل، مثل:

```
,4,
,6,7,
```

وهذا يسمح بربط الشاهد بأكثر من آية.

> ملاحظة تصميمية: هذا حقل متعدد القيم داخل نص واحد، وهو حل عملي لكنه غير مُطبّع (denormalized).

---

## العلاقات المنطقية

يرتبط هذا الجدول مع:

* جدول `book_quran` عبر `aya_index`
* جدول `quran_data` بشكل غير مباشر عبر الآيات

العلاقة الحالية تتم عبر الحقل النصي `aya_indices`.

---

## تحسينات مقترحة (اختيارية ومتقدمة)

### 1. إنشاء جدول ربط مُطبّع

بدلاً من تخزين عدة آيات في نص واحد:

```sql
CREATE TABLE shawahid_ayat (
    shawahid_id INTEGER,
    aya_index INTEGER
);
```

هذا يسمح بـ:

* استعلامات أسرع
* فهارس فعالة
* تكامل مرجعي أفضل

---

### 2. جدول مصادر الشواهد

```sql
CREATE TABLE shawahid_sources (
    code TEXT PRIMARY KEY,
    name TEXT
);
```

لتحويل رموز المصادر إلى تعريف رسمي.

---

### 3. فهرس بحث نصي

```sql
CREATE INDEX idx_shawahid_text
ON shawahid_entries(text);
```

لتحسين أداء البحث.

---

إذا رغبت، يمكننا الآن:

* تجميع خريطة العلاقات بين جداول القراءات والشواهد
* اقتراح نموذج بيانات مثالي للتطبيق
* تصميم طبقة استعلام فعالة للشواهد
* توحيد كل التوثيق في مستند مشروع شامل
