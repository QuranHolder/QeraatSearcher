## توثيق الجدول: `tagsmaster`

يمثل جدول **`tagsmaster`** جدولًا مرجعيًا لتعريف **وسوم (Tags) تصنيف اختلافات القراءات**. يُستخدم هذا الجدول لتوحيد أسماء الوسوم وتعريفها وربطها بفئات علمية وبالقراء المعنيين، ويُعد مرجعًا لعمود `tags` في جدول `quran_data`.

الجدول جزء من نظام **التصنيف الموضوعي** للقراءات.

---

## تعريف الجدول

```sql
CREATE TABLE "tagsmaster" (
    "id" INTEGER,
    "tag" TEXT UNIQUE,
    "description" TEXT,
    "qarees" TEXT,
    "category" TEXT,
    "srt" INTEGER,
    PRIMARY KEY("id")
);
```

---

## الهدف من الجدول

يوفر هذا الجدول:

* قائمة موحدة للوسوم المستخدمة في تصنيف القراءات
* وصفًا علميًا مختصرًا لكل وسم
* ربط الوسم بالقراء المعنيين (عند الحاجة)
* تجميع الوسوم ضمن فئات علمية
* ترتيبًا منطقيًا للعرض

ويُستخدم في:

* تصنيف سجلات جدول `quran_data`
* التصفية والبحث الموضوعي
* بناء قوائم تصنيف داخل التطبيق
* الإحصاءات والتحليلات

---

## أعمدة الجدول

### 1. id (INTEGER PRIMARY KEY)

معرّف رقمي فريد للوسم.

* المفتاح الأساسي للجدول
* يُستخدم داخليًا للربط

---

### 2. tag (TEXT UNIQUE)

الاسم الرمزي المختصر للوسم.

أمثلة:

* `idghamn`
* `ikhfaa`
* `imala`

يجب أن يكون:

* فريدًا
* ثابتًا
* مناسبًا للاستخدام البرمجي

يُستخدم كمرجع داخل جدول `quran_data`.

---

### 3. description (TEXT)

وصف عربي موجز للوسم.

أمثلة:

* الإدغام بغير غنة
* الإمالة
* تغليظ اللام

يُستخدم للعرض التوضيحي للمستخدم.

---

### 4. qarees (TEXT)

تحديد القراء المرتبطين بالوسم (إن وُجد).

أمثلة:

* ترك الغنة (لخلف)
* إخفاء (أبي جعفر)
* تغليظ اللامات (لورش)

قد يكون فارغًا إذا كان الوسم عامًا.

---

### 5. category (TEXT)

فئة علمية عامة ينتمي إليها الوسم.

أمثلة محتملة:

* إدغام
* إمالة
* مدود
* أحكام النون

يُستخدم لتجميع الوسوم في واجهة التطبيق.

---

### 6. srt (INTEGER)

رقم ترتيب العرض.

* يُستخدم لضبط ترتيب الوسوم في القوائم
* يسمح بترتيب منطقي ثابت

---

## مثال بيانات

| id | tag     | description            | qarees           | category | srt |
| -- | ------- | ---------------------- | ---------------- | -------- | --- |
| 18 | idghamn | الإدغام بغير غنة       | ترك الغنة (لخلف) | —        | 10  |
| 19 | ikhfaa  | إخفاء عند الخاء والغين | إخفاء (أبي جعفر) | —        | 11  |
| 20 | imala   | الإمالة                | الممال والمقلل   | —        | 5   |

---

## العلاقات المنطقية

يرتبط هذا الجدول مع:

* جدول `quran_data` عبر عمود `tags`

حيث تُخزن الوسوم هناك كنص مفصول بفواصل يشير إلى قيم `tag` في هذا الجدول.

---

## تحسينات مقترحة (اختيارية ومتقدمة)

### 1. جدول ربط مُطبّع بين السجلات والوسوم

بدلاً من تخزين الوسوم كنص:

```sql
CREATE TABLE quran_tags (
    quran_id INTEGER,
    tag_id INTEGER
);
```

هذا يحسن:

* أداء الاستعلام
* سلامة البيانات
* الفهرسة

---

### 2. فهرس ترتيب

```sql
CREATE INDEX idx_tags_sort
ON tagsmaster(srt);
```

---

### 3. تقنين الفئات

إنشاء جدول مرجعي للفئات:

```sql
CREATE TABLE tag_categories (
    id INTEGER PRIMARY KEY,
    name TEXT
);
```
